## Ansible Galaxy [(10:35)](https://youtu.be/FaXVZ60o8L8?t=635)

Shares and reuses roles via `ansible-galaxy` CLI.

- **`ansible-galaxy role install <role>`** — installs a role globally by default (`/etc/ansible/roles`)
- Use **`ansible.cfg`** to set `roles_path = ./roles` — keeps roles local to the project, avoids version conflicts between playbooks
- **`requirements.yml`** — lock role versions for repeatability:
```yaml
---
roles:
  - src: geerlingguy.homebrew
    version: 3.1.0
  - src: elliotweiser.osx-command-line-tools
    version: 2.3.0
```
- Install with **`ansible-galaxy install -r requirements.yml`** — downloads all listed roles into the configured `roles_path`
- **`ansible-galaxy role init <name>`** — scaffolds a new role skeleton (replaces deprecated `molecule init role`)

## Ansible Testing Spectrum [(17:17)](https://youtu.be/FaXVZ60o8L8?t=1037)

| Layer | Tool | What it checks |
|-------|------|----------------|
| 1 | `yamllint` | Valid YAML formatting (indentation, spacing) |
| 2 | `ansible-playbook --syntax-check` | Basic playbook structure, module resolution |
| 3 | `ansible-lint` | Best practices (naming tasks, avoiding shell when command works) |
| 4 | `molecule test` | Full integration test in a fresh environment |
| 5 | `--check` mode | Dry-run against production to preview changes |
| 6 | Parallel infra | Staging/prod-like environment rebuilt per deploy (rare) |

### Inline Testing

- **`debug`** — print variable values mid-playbook, supports `verbosity` to control visibility
- **`fail`** — halt execution (acts as a breakpoint when combined with `when`)
- **`assert`** — formal pass/fail based on conditions (e.g., file exists, port open, string in output)

### yamllint [(28:40)](https://youtu.be/FaXVZ60o8L8?t=1720)

- Install: `pip3 install yamllint`
- Run: `yamllint .`
- Default rules flag `yes`/`no` as truthy values (valid in Ansible). Tolerate via `.yamllint`:
```yaml
extends: default
rules:
  truthy:
    allowed-values: [true, false, yes, no]
```

### ansible-lint [(38:05)](https://youtu.be/FaXVZ60o8L8?t=2285)

- Install: `pip3 install ansible-lint`
- Run: `ansible-lint playbook.yml`
- **Common fixes it enforces:**
  - All tasks should be named
  - Prefer `command` over `shell` unless shell features are needed
  - Use `changed_when: false` for read-only commands (e.g., `uptime`)

## Molecule [(42:43)](https://youtu.be/FaXVZ60o8L8?t=2563)

Integration testing framework that spins up ephemeral instances (Docker/VMs), runs the role, verifies, then destroys.

### Setup

```bash
# Create a role first (molecule init role removed in v6.0.0)
ansible-galaxy role init myrole
cd myrole

# Create molecule scenario inside the role
molecule init scenario
```

This generates `molecule/default/`:

| File | Purpose |
|------|---------|
| `molecule.yml` | Driver, platform, provisioner config |
| `converge.yml` | Playbook that applies the role |
| `verify.yml` | Verification tests (Ansible or testinfra) |
| `create.yml` / `destroy.yml` | Lifecycle (auto-generated by Docker driver) |

### `molecule.yml`

```yaml
driver:
  name: docker
platforms:
  - name: instance
    image: geerlingguy/docker-ubuntu2404-ansible:latest
    pre_build_image: true
provisioner:
  name: ansible
```

### Test Matrix

Steps run in order by `molecule test`:

`dependency → cleanup → destroy → syntax → create → prepare → converge → idempotence → side_effect → verify → cleanup → destroy`

### Key Commands

| Command | What it does |
|---------|-------------|
| `molecule test` | Full lifecycle — fresh environment every time |
| `molecule converge` | Create + converge only — leaves container running |
| `molecule login` | SSH/exec into the running container (debugging) |
| `molecule verify` | Re-run the verifier against a running container |
| `molecule destroy` | Tear down all instances |
| `molecule list` | Show instance status |

### Development Workflow [(55:52)](https://youtu.be/FaXVZ60o8L8?t=3352)

Use `molecule converge` (not `test`) for iterative development — the container stays up, so you can inspect and re-run.

**Breakpoint debugging with `fail`:**
```yaml
- name: Breakpoint for debugging
  ansible.builtin.fail:
    msg: "Stopping for manual inspection"
```

The playbook halts, container stays alive, then `molecule login` drops you in to inspect files, services, etc. Remove the `fail` task and run `molecule converge` again to continue.

## Practical Gotchas

- **`molecule init role` removed in v6.0.0** — use `ansible-galaxy role init` then `molecule init scenario`
- **Docker driver needs `molecule-plugins[docker]`** — without it, Docker driver won't work
- **Stale apt cache in containers** — always use `update_cache: true` in apt tasks, otherwise "package not found" errors
- **Container creation can require many retries** (15+ on first run) — Molecule auto-retries with backoff
- **Role name in `converge.yml`** — when `meta/main.yml` has `namespace`, use the fully qualified name (e.g., `learning.myrole`); bare `myrole` won't resolve
- **Idempotence** — molecule runs converge twice automatically; a second `changed` means your role isn't idempotent
- **Cleanup** — always `molecule destroy` to avoid orphaned containers
